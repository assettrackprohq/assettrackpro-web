Step 4 — Which fields update existing FAR rows on Push?
The Barcode is always the pivot key — it identifies which row to update. Fields marked ✓ Update will overwrite the existing FAR value on push. Fields marked — Lock are write-once — only written when a row is first created, never overwritten (protects finance data like Net Book Value).
Step 5 — New assets not in FAR
When AssetTrackPro has assets that don't exist in the sheet (registered in the field but not yet in FAR), what should happen on Push?
Append — good for live deployments where field data should flow directly into the FAR.
Flag only — good for controlled FAR environments where finance must approve new additions.
Skip — good for audit-only mode where the FAR is master and AssetTrackPro only updates existing records.

Step 1 — Create Supabase Account

Go to supabase.com → Sign Up → Create New Project → choose a region close to your users (e.g. West Africa → use EU West). Note your Project URL and anon/public API key from Settings → API.
